# AI Development Rules

## Project

BuildTrustBG Platform

---

## General Rules

The AI must always:

- explain which files will be modified
- wait for approval before changing files
- use reusable React components
- follow Next.js best practices
- write clean TypeScript
- use modular architecture
- use Tailwind CSS consistently
- write production-ready code

---

## BuildTrustBG Rules

The AI must NEVER:

- invent companies
- invent reviews
- invent testimonials
- invent statistics
- invent years of experience
- invent customer data

Everything must be truthful.

---

## Design Rules

Always follow BuildTrustBG branding.

Primary colors:

- Orange
- Dark Blue
- White

The logo must never be modified.

The interface language must be Bulgarian.

---

## Development Workflow

Before every change:

1. Explain what will change.
2. List modified files.
3. Wait for approval.

After every change:

- Run npm run lint
- Run npm run build
- Fix all errors
- Explain completed work

---

## Architecture

Always create reusable components.

Never duplicate code.

Keep business logic separated from UI.

Prepare everything for future database integration.

---

## Long-Term Goal

BuildTrustBG is not a simple website.

It is a scalable marketplace platform for trusted construction companies in Bulgaria.
