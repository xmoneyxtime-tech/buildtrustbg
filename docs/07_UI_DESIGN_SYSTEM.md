# BTOS UI Design System

## Purpose
Define visual and interaction standards for trust-centric, conversion-oriented interfaces.

## Current Status
Active baseline with Tailwind-driven reusable components; further formalization in progress.

## Design Principles
- Clear trust signals and transparent information hierarchy
- Responsive layouts across mobile and desktop
- Accessible interactions and readable typography
- Consistent spacing and component behavior
- Bulgarian-first UX with clear action paths

## UI System Guidelines
- Reuse shared components before creating new variants
- Keep CTA hierarchy clear for conversion
- Use semantic HTML and accessible labels
- Maintain consistent color usage for trust and status

## Future Roadmap
- Formal component inventory and token catalog
- Trust signal visual language and badge standards
- Dashboard KPI and recommendation pattern library
